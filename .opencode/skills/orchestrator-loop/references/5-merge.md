## Stage 5: MERGE

Create a branch, commit, push, and open a PR.

| Step | Tool | What it does |
|---|---|---|
| 5a | `bash`("git checkout -b {prefix}N-short-name") | Create branch |
| 5b | `bash`("git add -A && git commit -m 'msg (#N)'") | Commit with issue reference |
| 5c | `bash`("git push origin HEAD") | Push |
| 5d | `gh-pr_create`(title, body, head, base="main") | Open PR with description and issue reference |

### PR body template

```
Closes #<N>

This PR was auto-generated by the orchestrator-loop v2.
Human review and approval required before merging.
```

### Branch prefix convention

Use the issue label to determine prefix:
- `bug` or `bugfix` label → `bugfix/N-short-name`
- `feature` or `enhancement` → `feature/N-short-name`
- `docs` → `docs/N-short-name`
- Default → `chore/N-short-name`
